# KMP Pre-Migration Coverage Inventory

This inventory turns the KMP migration goal into concrete coverage obligations. Each row names the current Android and iOS test evidence that should be preserved or extended before the corresponding behavior is moved into shared KMP code.

## Coverage Status Legend

- `Covered`: both Android and iOS have direct tests for the behavior family.
- `Partial`: one platform has direct tests or both platforms have tests but important edge cases still need golden vectors.
- `Missing`: no direct current test was found in the repository inspection.
- `Not assessed`: the behavior requires deeper source inspection before assigning coverage.

## Platform-Owned Migration Boundary

These rows are intentionally not common-migration-ready during pre-migration coverage work: `BLE device session lifecycle`, `GATT clients`, `Android Bluedroid host behavior`, and `iOS CoreBluetooth host behavior`. They remain platform-owned until a future behavior slice extracts a pure codec or deterministic state machine with explicit common fake-transport tests, Android facade compatibility tests, iOS facade compatibility tests, and a row update in this inventory. Android Bluedroid unknown/dead GATT cleanup is pinned as host-owned behavior in `GattCallbackTest.kt` and must not move to shared KMP runtime code. The policy gates reject removing this boundary or changing those rows to migration-ready language without adding a stronger shared contract.

## Full-Coverage Exit Criteria Before Migration

- Every row marked `Partial` has either new tests, documented platform-specific ownership, or a migration deferral note.
- Every parser that moves to KMP has shared golden vectors covering valid, invalid, empty, boundary, and unknown-value cases.
- Android and iOS characterization tests use the same vectors or prove equivalent expected behavior.
- Public facade tests prove Android and iOS APIs keep current semantics after shared delegation.
- Hardware-dependent tests are limited to smoke coverage; deterministic behavior is covered without physical devices.
- Validation commands are documented and runnable for the migrated slice.
