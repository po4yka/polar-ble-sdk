# KMP Fake Transport Test Plan

This plan defines the fake-transport coverage required before moving connection-independent runtime behavior or public facade delegation into KMP. Parser and model migrations may proceed with golden vectors first, but runtime orchestration must not move until the relevant fake-transport rows below are implemented on Android, iOS, and KMP common tests where shared behavior is intended.

## Scope Boundary

Android Bluedroid and iOS CoreBluetooth scanning, bonding, permission, adapter, background, and OS callback behavior remain platform-specific. KMP common code may own only deterministic orchestration that can be expressed through a fake transport: command sequencing, read/write request routing, notification filtering, stream lifetime, cancellation, timeout, retry policy, and disconnect propagation.

## Fake Transport Contract

| Capability | Required controls | Required observations | KMP ownership |
|---|---|---|---|
| Connection state | Start connected, start disconnected, disconnect after N operations, reconnect after failure | Calls attempted, terminal error, cleanup callback count | Shared only for connection-independent command guards |
| Service readiness | Services available, missing service, delayed readiness | Feature readiness result, wait ordering, timeout path | Shared if feature readiness is extracted |
| Reads | Return bytes, throw transport error, delay response, never respond | Request path/UUID, decoded model, mapped error | Shared for pure request/response codecs |
| Writes | Accept bytes, throw transport error, capture payload, delay completion | Byte-for-byte payload, completion/error ordering | Shared for command builders and serializers |
| Notifications | Emit ordered values, emit unknown values, emit after completion, complete, fail, never emit | Filtered emissions, late-error behavior, duplicate completion behavior | Shared for stream orchestration that is not OS-specific |
| Time | Virtual timeout clock, retry delay clock, cancellation before delay finishes | Retry count, timeout error, no leaked observer | Shared for timeout/retry policy |

## Pre-Migration Gates

- A runtime migration slice must name the exact rows from the matrix that it implements.
- A slice that delegates only parser/model code does not need fake transport tests unless it changes stream or facade behavior.
- A slice that delegates public facade operations must include both platform facade tests and common fake-transport tests for success, error, cancellation, and timeout where applicable.
- A slice that keeps a transport area platform-specific must update `KmpCoverageInventory.md` with the ownership reason and the existing platform tests that remain authoritative.
- A runtime test must assert observer cleanup or cancellation propagation whenever it opens a stream, registers a listener, or starts an internal task.

## PSFTP Runtime Harness Requirements

`PsFtpByteCodecCommonPolicyTest.kt` owns the pure RFC76 frame-header decoding, RFC60 complete-message-stream encoding, and RFC76 MTU splitting vectors before runtime orchestration moves; the fake-transport harness below should build on those byte-codec decisions instead of duplicating header-bit assertions in every runtime test.

The PSFTP runtime migration needs a dedicated fake-transport harness before write orchestration or wait-notification orchestration moves to common code. The harness must expose independent MTU and D2H channels, explicit descriptor-enabled state, controllable transmit success/failure, controllable write acknowledgements, and a virtual protocol clock. Directly collecting Android `waitForNotification()` repeatedly is not a sufficient substitute because current platform flow cancellation timing can block tests outside a fake runtime boundary.

The first common harness slice should consume the existing PSFTP vectors in this order: request response reassembly, request response error policy, write success progress, write interruption error policy, write transport failure policy, notification reassembly, notification error policy, initial-silence timeout policy, notification ordering, notification continuation timeout policy, and write acknowledgement timeout policy. Timeout vectors must advance virtual time rather than waiting for platform protocol constants. Android and iOS tests now guard the timeout planning vectors so `notification-continuation-timeout-policy` and `write-ack-timeout-policy` cannot be treated as completed platform runtime coverage until fake-clock or injectable-timeout execution exists; Android now executes both timeout vectors through an injected short `protocolTimeoutSeconds`, and linked iOS now executes both timeout vectors through an injected short `PROTOCOL_TIMEOUT`. The former Android-hosted `PsFtpCommonFakeRuntimeTest` prototype has been retired because `PsFtpRuntimePolicyCommonTest.kt` now consumes the same deterministic subset as executable shared `commonTest` production-planner coverage: request response reassembly, typed response errors, write progress platform-split pinning, write interruption as typed response error, write transport failure, notification reassembly, notification ordering, notification error policy split pinning, initial-silence no-emission, consumer-owned virtual-clock timeout cleanup, continuation timeout, and write acknowledgement timeout. `PsFtpRuntimePolicyCommonTest.kt` now also includes a PSFTP-specific common fake harness for the timeout subset, asserting independent MTU writes, independent D2H packets, descriptor-enabled state, write acknowledgement count, scanner pause/resume cleanup, active observer cleanup, and no leaked pending operation after initial-silence timeout, continuation timeout, or write-ack timeout. Platform facade timeout compatibility is now pinned before production PSFTP runtime moves to common code; a future shared runtime still needs fake-clock facade compatibility before production PSFTP timeout delegation if timeout scheduling itself moves into common orchestration.

Required PSFTP harness observations are: emitted notification order, terminal error type before facade mapping, payload bytes written to MTU/H2D channels, write acknowledgement count, scanner pause/resume or equivalent operation-scope cleanup, cancellation cleanup after a collector stops, and no leaked pending operation after timeout.
