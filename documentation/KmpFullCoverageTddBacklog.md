# KMP Full-Coverage TDD Backlog

This backlog turns the pre-migration coverage inventory into executable TDD work. Use it after reading `KmpCoverageInventory.md`, `KmpTddStrategy.md`, `KmpFakeTransportTestPlan.md`, and `KmpValidationCommands.md`; the purpose is to choose the next failing-test slice without rerunning broad Gradle tasks after every small edit.

## Operating Rules

- Start each slice by adding or tightening characterization tests and golden vectors before moving production code.
- Prefer shared `commonTest` policy tests for pure byte parsing, model mapping, deterministic request planning, and fake transport runtime state.
- Keep Android-only and iOS-only BLE host behavior out of common code unless a slice first defines a platform-neutral contract.
- Do not migrate a behavior family while its row in `KmpCoverageInventory.md` still says `Partial` without a concrete deferral or a failing shared/common test.
- Treat every `platformExpectations.commonDecision`, top-level `commonDecision`, and `migrationOwnership` field as an explicit TDD decision point rather than background documentation.
- Batch several file/test/doc edits before Gradle validation so the migration does not stall on repeated manual app prompts.
